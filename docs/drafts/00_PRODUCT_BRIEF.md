# Product Brief: Dore

## 한 줄 정의

`Dore`는 사용자의 일상, 지식, 업무, 개발, 투자 활동을 장기적으로 기억하고 정리하며, 필요한 시점에 제안하고 실제 작업까지 수행하는 개인 AI 비서다.

## 목표

- 사용자의 일상과 업무 맥락을 장기 기억으로 축적한다.
- 흩어진 정보를 정리하고 검색 가능한 개인 지식 체계로 만든다.
- 사용자가 놓치기 쉬운 일정, 결정, 작업, 리스크를 적절한 시점에 알려준다.
- 매일 오전 6시 KST에 개인/업무/개발/투자 정보를 모아 briefing한다.
- 소프트웨어 엔지니어 역할을 수행한다.
- 필요한 기능을 스스로 설계, 구현, 테스트, 개선한다.
- LLM 토큰 사용량과 비용을 최적화한다.
- API 기반 주식 트레이딩을 수행한다.
- 기존 `hermes-agent`가 제공하는 기능을 동일하게 수행한다.
- 로컬 상시 실행, 메신저 봇, 데스크톱 앱 형태로 사용할 수 있다.

## 사용자

- 단일 사용자: 프로젝트 소유자 본인. 프로젝트를 설치하는 사용자.
- 기본 언어: 한국어.
- 필요 시 영어 문서, 코드, API 문서를 읽고 요약/변환할 수 있어야 한다.
- 온보딩 과정 필요 (openclaw or hermes-agent 유사기능)

## 핵심 가치

- 기억: 대화와 파일이 사라지지 않고 지속적으로 축적된다.
- 정리: 원본 자료, 요약, 판단, 작업 이력이 분리되어 관리된다.
- 실행: 단순 조언이 아니라 실제 파일 작업, 코드 작업, API 호출, 자동화 실행까지 수행한다.
- 선제성: 사용자가 묻기 전에 필요한 정보를 제안한다.
- 검증: 중요한 작업은 테스트, 로그, 근거, 재현 가능한 절차를 남긴다.
- 비용 효율: 모든 정보를 매번 LLM에 넣지 않고, 장기 기억과 요약 계층을 활용한다.

## 주요 기능 범위

### 개인 비서

- 일정, 할 일, 아이디어, 메모, 관심사, 목표 관리.
- 대화와 작업 이력 기반 컨텍스트 유지.
- 필요한 시점에 리마인드와 제안 제공.
- 파일, 문서, 웹 자료를 정리하여 개인 지식 베이스로 축적.

### 지식 관리

- `refs/llm-wiki.md`의 철학을 반영한 지속형 개인 위키 운영.
- 원본 자료는 보존하고, Agent가 요약/연결/갱신하는 wiki 계층을 관리.
- 질문에 대한 답변 중 가치 있는 결과는 wiki에 다시 저장.

### 소프트웨어 엔지니어

- 요구사항 정리.
- 기획 문서, 설계 문서 작성.
- 코드 구현.
- 코드 리뷰.
- 테스트 작성 및 실행.
- 버그 분석.
- 리팩터링.
- 배포/운영 지원.
- 필요한 내부 도구를 직접 제작.

### 주식 트레이더

- 사용자 제공 API를 통해 시장 데이터 조회.
- 포트폴리오와 보유 종목 추적.
- 전략 신호 생성.
- 백테스트.
- 주문 실행.
- 손실 제한, 주문 제한, 기록 관리.
- 트레이딩 세부 정책은 [17_TRADING_POLICY_DRAFT.md](17_TRADING_POLICY_DRAFT.md), [18_RISK_AND_COST_DEFAULTS.md](18_RISK_AND_COST_DEFAULTS.md), [22_CONFIG_SCHEMA_DRAFT.md](22_CONFIG_SCHEMA_DRAFT.md)를 기본값으로 사용한다.
- 대상 시장은 국내 주식과 미국 주식.
- 자동 주문은 공식 API가 확인되는 증권사만 대상으로 한다.
- 토스증권을 1차 connector 후보로, 신한증권을 2차 connector 후보로 둔다.
- 삼성증권은 공식 개인용 Open API 확인 전까지 자동 주문 대상에서 제외한다.

### 실행 인터페이스

- 로컬 상시 실행 daemon.
- Telegram bot.
- 데스크톱 앱.
- 필요 시 CLI/TUI.

## 비범위 또는 별도 결정 필요

- 사용자가 명시하지 않은 금융상품 자동 거래.
- 법률/세무/의료 자문을 확정적 조언으로 제공하는 행위.
- 출처 없는 투자 추천.
- 사용자 승인 없이 위험도가 높은 시스템 변경.
- 사용자 승인 없이 대규모 금전 손실 가능성이 있는 주문 실행.

## 성공 기준

- 사용자가 "내가 전에 말한 것"을 다시 설명하지 않아도 Agent가 맥락을 회복한다.
- Agent가 작업 결과를 문서, 코드, 로그, 메모로 남긴다.
- Agent가 필요한 정보를 필요 시점에 먼저 제안한다.
- 개발 작업은 테스트 또는 검증 절차와 함께 완료된다.
- 트레이딩 작업은 모든 주문과 판단 근거가 기록된다.
- 실제 주문 전 최소 30일 paper trading/dry-run journal이 축적된다.
- 장기 운영 시 토큰 비용이 선형적으로 폭증하지 않는다.
- 로컬, 메신저, 데스크톱 앱 사이에서 세션과 기억이 이어진다.
