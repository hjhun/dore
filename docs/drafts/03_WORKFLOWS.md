# Workflows

## 1. 일상 기억 저장

### 입력

- 사용자 대화.
- 메모.
- 파일.
- 일정.
- 웹 자료.

### 처리

1. 장기 기억 가치 판단.
2. 저장 위치 결정.
3. 관련 wiki 페이지 업데이트.
4. 필요한 경우 할 일 또는 리마인더 생성.
5. 로그 기록.

### 출력

- 업데이트된 wiki.
- 작업 로그.
- 필요한 제안.

## 2. 필요 시점 제안

### 트리거

- 매일 오전 6시 KST daily briefing.
- 일정 도래.
- 미완료 작업 장기 방치.
- 새 정보와 기존 목표의 연결 발견.
- 투자 신호 발생.
- 개발 프로젝트의 다음 단계가 명확해짐.

### 처리

1. 현재 시간과 활성 컨텍스트 확인.
2. 관련 memory 검색.
3. 제안 후보 생성.
4. 중요도와 긴급도 평가.
5. 사용자에게 간결하게 제안.

### 출력 예시

- "오늘 오전 6시 briefing".
- "오늘 확인해야 할 3가지".
- "이 작업은 자동화할 수 있습니다".
- "이 종목은 전략 조건 A/B를 만족했습니다".
- "이 프로젝트는 테스트가 아직 없습니다".

## 3. 개발 작업 수행

### 입력

- 사용자 요청.
- 기존 코드.
- 설계 문서.
- 이슈/버그 설명.

### 처리

1. 저장소 구조 파악.
2. 요구사항 정리.
3. 설계 또는 변경 계획 작성.
4. 코드 수정.
5. 테스트/검증 실행.
6. 결과 요약.
7. 관련 문서와 memory 업데이트.

### 출력

- 변경된 코드.
- 테스트 결과.
- 변경 요약.
- 후속 작업.

## 4. 코드 리뷰

### 입력

- diff.
- PR.
- 파일 경로.
- 사용자 지정 리뷰 관점.

### 처리

1. 변경 의도 파악.
2. 버그, 회귀, 보안, 테스트 누락 중심 검토.
3. 파일/라인 기준으로 지적.
4. 개선안 제시.

### 출력

- 심각도 순 리뷰 코멘트.
- 테스트 갭.
- 필요한 질문.

## 5. 자기 기능 개선

### 트리거

- 반복 작업 발견.
- 사용자가 새 기능 요청.
- Agent가 현재 기능 부족을 식별.
- 실패 로그가 누적됨.

### 처리

1. 기능 명세 작성.
2. 위험도 평가.
3. 구현.
4. 테스트.
5. 문서 업데이트.
6. 사용 로그 기반 개선.

### 출력

- 새 기능 또는 보완된 기능.
- 변경 로그.
- 검증 결과.

## 6. 주식 트레이딩

### 입력

- 시장 데이터 API.
- 계좌/포트폴리오 API.
- 주문 API.
- 사용자 전략.
- 리스크 제한.
- 설정 파일 또는 사용자가 Agent에게 전달한 정책 정보.

### 처리

1. 공식 API 지원 여부 확인.
2. 데이터 수집.
3. 전략 조건 평가.
4. 리스크 제한 확인.
5. 주문 후보 생성.
6. dry-run 또는 paper trading journal 기록.
7. 승인 정책 확인.
8. 주문 실행.
9. 체결 결과 확인.
10. 거래 로그 기록.

리스크 제한이 아직 설정되지 않았으면 실제 주문은 중단하고 필요한 설정 항목을 요청한다.

MVP에서는 실제 주문을 기본 비활성화하고, trading watch와 dry-run을 우선한다.

### 출력

- 시장 요약.
- 전략 신호.
- 주문 내역.
- 체결 내역.
- 손익/리스크 리포트.

## 8. Daily 6 AM Briefing

### 입력

- 개인 일정과 할 일.
- 활성 프로젝트.
- 전일/당일 개발 상태.
- 국내 증시 정보.
- 미국 증시 마감 정보.
- 관심 종목과 포트폴리오.
- token/cost usage.
- memory 업데이트.

### 처리

1. source collector가 구조화 데이터를 수집한다.
2. 전일 briefing과 변경분을 비교한다.
3. 저비용 모델로 초안 요약을 만든다.
4. 필요 시 고성능 모델로 최종 briefing을 다듬는다.
5. Telegram으로 짧은 요약을 보낸다.
6. Electron Dashboard에 상세 briefing을 표시한다.
7. memory log에 저장한다.

### 출력

- Telegram morning summary.
- Dashboard 상세 briefing.
- `memory/logs/daily/YYYY-MM-DD.md`.
