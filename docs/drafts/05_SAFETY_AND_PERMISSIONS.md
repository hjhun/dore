# Safety and Permissions

## 목적

Agent는 실제 작업을 수행해야 하지만, 개인 정보, 시스템 권한, 금융 거래를 다루므로 명확한 안전 정책이 필요하다.

## 기본 원칙

- 모든 고위험 작업은 기록한다.
- 파괴적 작업은 사전 검증한다.
- 금융 거래는 전략, 리스크, 주문 결과를 모두 기록한다.
- 개인 정보와 자격증명은 최소 노출 원칙을 따른다.
- Agent가 모르는 내용은 추측으로 실행하지 않는다.

## 권한 등급

### Read

- 파일/문서/API 조회.
- 시장 데이터 조회.
- 계좌 잔고 조회.

### Write

- 문서 작성.
- wiki 수정.
- 코드 수정.
- 할 일/일정 생성.

### Execute

- 스크립트 실행.
- 테스트 실행.
- 배포.
- 외부 API mutation.

### Trade

- 주문 생성.
- 주문 취소.
- 주문 정정.
- 포지션 청산.

## 트레이딩 리스크 제한 초안

아래 값은 [18_RISK_AND_COST_DEFAULTS.md](18_RISK_AND_COST_DEFAULTS.md)의 기본값을 사용한다. 사용자가 계좌 규모와 투자 성향에 맞춰 별도 설정하면 그 설정이 우선한다.

- 1회 주문 최대 금액.
- 1일 최대 손실.
- 종목당 최대 비중.
- 전체 현금 보유 최소 비율.
- 레버리지 사용 여부.
- 공매도/파생상품 허용 여부.
- 장중 자동매매 허용 시간.
- 손절/익절 기준.
- 뉴스/공시 기반 거래 허용 여부.

리스크 제한이 비어 있거나 불완전하면 실제 주문은 실행하지 않는다. 이 경우 Agent는 주문 후보, 필요한 누락 설정, 예상 리스크만 보고한다.

## Trading 기본 안전 결정

- MVP에서는 실제 주문을 기본 비활성화한다.
- 공식 API가 확인되는 증권사만 자동 주문 대상으로 삼는다.
- 토스증권은 1차 connector 후보, 신한증권은 2차 connector 후보로 둔다.
- 삼성증권은 공식 개인용 Open API 확인 전까지 자동 주문 대상에서 제외한다.
- RPA/GUI 자동 주문은 기본 금지한다.
- paper trading 또는 dry-run journal을 먼저 축적한다.
- 레버리지, 공매도, 파생상품, 신용/미수 거래는 기본 금지한다.

## 중단 조건

Agent는 아래 상황에서 실행을 중단하고 사용자에게 확인해야 한다.

- API 응답이 예상 스키마와 다름.
- 계좌/잔고/주문 가능 금액이 불명확함.
- 주문 수량 계산에 오류 가능성이 있음.
- 손실 제한을 초과할 가능성이 있음.
- 시스템 파일 삭제 또는 대량 수정이 필요함.
- 보안 자격증명 변경이 필요함.
- 사용자의 기존 지시와 충돌하는 요청이 발생함.
