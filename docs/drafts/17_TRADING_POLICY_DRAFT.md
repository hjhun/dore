# Trading Policy Draft

## 목적

Dore가 국내 주식과 미국 주식을 다루기 위한 기본 정책을 정의한다.

사용자의 목표는 자산을 크게 늘리는 것이지만, Dore는 손실 가능성을 낮추고 검증 가능한 방식으로 트레이딩 기능을 확장해야 한다.

## 기본 원칙

- 공식 API가 있는 증권사만 자동 주문 대상으로 삼는다.
- 실제 주문 전 paper trading 또는 dry-run 검증을 거친다.
- 실제 주문은 별도 활성화 전까지 비활성화한다.
- 레버리지, 공매도, 파생상품은 기본 금지한다.
- 모든 주문 후보, 판단 근거, 실행 결과를 기록한다.
- 수익 보장을 암시하지 않는다.
- 손실 제한이 설정되지 않으면 실제 주문하지 않는다.

## 대상 시장

- 국내 주식.
- 미국 주식.
- ETF.

MVP 제외:

- 선물.
- 옵션.
- ELW.
- CFD.
- 신용/미수.
- 레버리지 ETF 자동매매.
- 코인.

## 증권사 우선순위

### 1. 토스증권

우선순위: 높음.

이유:

- 토스증권 공식 Open API 페이지가 확인된다.
- 국내주식과 해외주식 주문, 계좌 조회, 보유 종목, 주문 가능 금액 같은 기능을 제공하는 방향으로 안내되어 있다.

정책:

- Dore의 1차 자동매매 connector 후보로 둔다.
- 공식 문서, 약관, 신청 가능 여부, sandbox 여부를 확인한 뒤 구현한다.
- 실제 주문 전 dry-run journal을 먼저 만든다.

### 2. 신한증권

우선순위: 중간.

이유:

- 신한 Open API 포털과 신한i indi 관련 공식 채널이 확인된다.
- 다만 Dore MVP에서는 실제 주문 API 범위, Windows 의존성, 자동화 적합성을 별도로 검증해야 한다.

정책:

- 2차 connector 후보로 둔다.
- 계좌/시세/주문 가능 범위를 공식 문서 기준으로 재확인한다.
- 로컬 Windows 환경과의 궁합을 검토한다.

### 3. 삼성증권

우선순위: 낮음.

이유:

- 현재 확인 가능한 공식 삼성증권 자료는 HTS/DTS/Web trading 중심이다.
- 공식 개인용 Open API가 확인되기 전까지 자동 주문 대상으로 보지 않는다.
- 삼성증권은 자동주문 화면/약정 기능은 있으나, Dore가 직접 주문 API로 제어하는 근거로 사용하지 않는다.

정책:

- MVP에서는 삼성증권 자동 주문 제외.
- 보유 계좌 리포트, 수동 참고, 웹/HTS 정보 확인 정도로 제한한다.
- RPA/GUI 자동 주문은 보안/약관/유지보수 리스크가 커서 기본 금지한다.

## 자동 주문 활성화 조건

아래 조건을 모두 만족해야 실제 주문을 활성화한다.

1. 공식 API 문서 확인.
2. API 약관 확인.
3. 계좌 권한 확인.
4. sandbox 또는 paper trading 검증.
5. 최소 30일 dry-run journal.
6. 손실 제한 설정.
7. 1회 주문 최대 금액 설정.
8. 종목당 최대 비중 설정.
9. 사용자 명시 승인.
10. kill switch 구현.

## 기본 리스크 제한

사용자가 별도 설정하기 전 기본값은 [18_RISK_AND_COST_DEFAULTS.md](18_RISK_AND_COST_DEFAULTS.md)를 따른다.

MVP 기본값:

- 실제 주문: 비활성화.
- paper trading: 활성화 가능.
- 1회 주문 최대 금액: 실제주문 비활성화 상태에서는 적용하지 않음.
- 1일 최대 손실: 실제주문 비활성화 상태에서는 적용하지 않음.
- 종목당 최대 비중: 실제주문 비활성화 상태에서는 적용하지 않음.
- 레버리지/공매도/파생상품: 금지.
- 장중 자동매매: 비활성화.

파일럿 실제주문 활성화 시 기본값:

- 1회 주문 최대 금액: `min(NAV * 1%, 100,000 KRW equivalent)`.
- 1일 신규 매수 총액: `min(NAV * 3%, 300,000 KRW equivalent)`.
- 1일 손실 한도: `min(NAV * 1%, 100,000 KRW equivalent)`.
- 종목당 최대 비중: `NAV * 10%`.
- 신규 종목 첫 진입 비중: `NAV * 2%` 이하.
- 현금 최소 보유 비율: `NAV * 20%`.

계좌 평가액을 확인할 수 없거나 값이 계산되지 않으면 실제 주문하지 않는다.

## 전략 개발

Dore는 전략을 제안할 수 있다.

전략의 lifecycle과 signal schema는 [25_TRADING_STRATEGY_FRAMEWORK.md](25_TRADING_STRATEGY_FRAMEWORK.md)를 따른다.

허용:

- 모멘텀 watch.
- 리밸런싱 후보.
- 손절/익절 규칙 후보.
- 이벤트 기반 watch.
- 장기 투자 아이디어 정리.
- 백테스트.

금지:

- 근거 없는 확정 수익 주장.
- 손실 제한 없는 자동 진입.
- 뉴스 한 줄만 보고 즉시 주문.
- 과도한 단기 매매를 기본 전략으로 삼는 것.
